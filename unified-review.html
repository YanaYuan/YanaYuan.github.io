Unified Review

心跳的另一面——用自然对话帮你过一遍所有项目的待决策事项，确认即执行。

concordia-pmo-manager / skills / unified-review

核心概念

Unified Heartbeat 在后台自动运行：扫描信息、思考下一步、执行白名单内的操作。但有些事情它不能自己做——需要你拍板。这些事会被记录下来，等你来 Review。

Review 的本质：把心跳积累的待决策事项，一个一个摆到你面前，你说一句话就执行。

Heartbeat（自动，每小时） → 白名单允许 → 直接执行
Heartbeat（自动，每小时）→ 白名单不允许 → 写入日志等你 Review

Review（你触发） → 你 approve → 当场执行

不存在"approve 了但留到下次再做"的情况。你在场就是最高权限，确认即执行。

对话流程

1

简报

2-3 句话概括：心跳发现了什么、有什么风险、接下来有几个决策要过。纯信息，不需要你做任何事。

2

资料收集决策（Phase A）

心跳扫到的相关素材：要不要收进项目资料库？一条一条来，快速 yes/no。建议不收的自动跳过。

3

执行决策（Phase B）

心跳想做但没权限的操作：给团队发消息、创建任务、对齐 scope 等。每条带建议和风险。

4

下一个项目

当前项目过完后自然过渡到下一个。所有项目处理完给个 3-5 行总结。

每个决策完成后立即持久化——session 断了也不丢。

两个阶段

Phase A — 资料收集

心跳扫到的相关素材（会议纪要、团队讨论、文档等），快速决定收不收。建议不收的自动跳过，你只看建议收的。

Phase B — 执行决策

需要更多思考的操作：创建任务并分配、给团队发消息、对齐 scope、审批交付物。每条带风险评估。

它读取什么

Review 独立于心跳——即使隔了一天，也能自主获取完整上下文：

项目定义

目标、团队、deadline

你的待办

当前任务和状态

Planner 任务

已分配的任务和进度

心跳日志

上次 review 后的所有发现

对话风格

DO

  • 像同事给 brief，不像系统报告
  • 每条都有建议、理由、风险
  • 用业务语言，附原链接和参与人
  • 支持批量操作和随时打断

DON'T

  • 暴露文件路径和系统 ID
  • 一次给多个决策
  • 倒一大堆内容摘要
  • 问"先看哪个项目？"——直接从最紧急的开始

设计原则

Approve 即执行 你在场 = 最高权限，不延迟
即时持久化 每个决策立即写入，断点安全
永远有观点 每条决策带明确建议，不中立陈列
Heartbeat 闭环 已 review 的不再重复提
