System Architecture Overview

Concordia Project Agent System

An AI-native project management framework where a Project Agent coordinates multiple Member Agents to plan, execute, and deliver complex projects — with humans in the loop for decisions and review.

01

Three-Phase Lifecycle

Every project moves through three distinct phases: initialization, planning, and a continuous execution loop with daily and hourly rhythms.

Phase 1 — Initialization

Setup

  • project-init once
  • source-collection once
  • todo-breakdown once
Phase 2 — Project Loop

Manage

  • project-heartbeat hourly
  • project-dashboard post-heartbeat
  • todo-update daily
  • project-knowhow daily
Phase 3 — Member Loop

Execute

  • member-heartbeat hourly

Each team member has a dedicated Member Agent that executes tasks, produces deliverables, and manages async review with the human owner.

Four Stakeholders

AI Agent

Project Agent

The manager. Scans the environment, assigns tasks, tracks progress, reviews deliverables, discovers new work, escalates risks. Never produces deliverables directly.

AI Agent

Member Agent

The executor. One per team member. Deeply understands tasks, does the actual work (documents, code, research, analysis), submits for review, delivers approved results.

Human

Project Owner

Initiates the project. Confirms plans, approves task breakdowns, receives risk escalations, makes strategic decisions when the Project Agent flags uncertainty.

Human

Human Owner

The team member. Reviews deliverables produced by their Member Agent, provides feedback or approval, gets asked for help when the agent is stuck.

02

Skill Matrix

Eight skills form the complete system. Each has clearly defined inputs, outputs, interactions, and trigger conditions.

Skill Input What It Does Output Interaction Trigger
project-init User verbal input Guided conversation to collect project goals, deliverables, success criteria, timeline, and team project.yaml Project Owner multi-turn dialog User says "new project"
source-collection project.yaml Read-all-then-judge: reads 2 weeks of emails, Teams, meetings + searches SharePoint & Web Sources/ + _index.md None After project.yaml created
todo-breakdown project.yaml + Sources/ Meta-thinking self-audit → decompose tasks → assign owners → user confirms todos.md Project Owner confirms plan After source-collection
project-heartbeat project.yaml + todos.md + latest 2 heartbeats + real-time scan (Teams, email, calendar, meetings) Scan new content → add todos → assign / start / inquire / review → risk escalation Updated todos.md + Sources/ + heartbeats/*.md Member Agent assign/track · Project Owner escalation Hourly weekdays
project-dashboard project.yaml + todos.md + latest 2 heartbeats Generate full project status report dashboard.md None Post-heartbeat / manual
todo-update project.yaml + todos.md + heartbeats + broad scan (Teams, email, calendar, meetings, Web, docs) Collision: information meets goals → discover blind spots & opportunities → update todos Updated todos.md + heartbeats/*_todo-review.md Project Owner for deprecated tasks Daily
project-knowhow project.yaml + todos.md + today's heartbeats + knowhow.md Reflect on events → distill decisions, lessons, patterns, people insights, tools → ADD / REPLACE / EVICT knowhow.md (iterative single file) None Daily
member-heartbeat project.yaml + Sources/ + project todos.md + project heartbeats + member-todos.md + member heartbeats + owner comms (Teams, email, calendar, meetings) + Project Agent channel Deep thinking → status-driven action: execute task / revise per feedback / deliver approved result; ask Project Agent when unclear, ask owner when stuck Updated member-todos.md + deliverables/<task-id>/ + member-heartbeats/*.md Project Agent ask/deliver · Human Owner review/help Hourly weekdays
03

File Registry

Every file in the system has a clear owner (who writes it) and consumers (who reads it). Nothing is ambiguous.

Project Definition

project.yaml Written by project-init · Read by all

Task Management

todos.md Written by todo-breakdown, project-heartbeat, todo-update
member-todos.md Written by member-heartbeat

Sources/

_index.md Master index of all sources
emails/ Email thread archives
teams/ Teams conversation archives
meetings/ Meeting transcripts & recaps
sharepoint/ SharePoint / OneDrive docs
web/ Web research results

Logs & Reports

heartbeats/*.md Project Agent heartbeat logs
member-heartbeats/*.md Member Agent heartbeat logs
dashboard.md Project status report
knowhow.md Accumulated project knowledge

Deliverables

deliverables/<task-id>/ Member Agent produces · Project Agent reviews · contains _submission.md with revision history
