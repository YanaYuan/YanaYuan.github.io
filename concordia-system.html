系统架构概览

Concordia 项目 Agent 系统

AI 原生的项目管理框架：Project Agent 协调多个 Member Agent 完成项目的规划、执行与交付，人类在关键决策和 Review 环节保持控制。

01

三阶段生命周期

每个项目经历三个阶段：初始化、规划，以及以小时和天为节奏的持续执行循环。

阶段一 — 初始化

启动

  • project-init 一次性
  • source-collection 一次性
  • todo-breakdown 一次性
阶段二 — 项目循环

管理

  • project-heartbeat 每小时
  • project-dashboard 心跳后
  • todo-update 每天
  • project-knowhow 每天
阶段三 — 成员循环

执行

  • member-heartbeat 每小时

每位团队成员有专属的 Member Agent，负责执行任务、产出交付物、与人类 Owner 异步 Review。

四个利益相关方

AI AGENT

Project Agent

管理者。扫描环境、分配任务、追踪进度、验收交付物、发现新工作、上报风险。不直接产出交付物。

AI AGENT

Member Agent

执行者。每位成员一个。深度理解任务，实际完成工作（文档、代码、调研、分析），提交 Review，交付已批准的成果。

人类

Project Owner

项目发起人。确认计划、审批任务拆解、接收风险上报、在 Project Agent 标记不确定性时做出战略决策。

人类

Human Owner

团队成员本人。Review Member Agent 产出的交付物，提供修改反馈或批准，在 Agent 遇到困难时提供帮助。

通信流向

02

技能矩阵

八个技能构成完整系统。每个技能都有明确定义的输入、输出、交互方式和触发条件。

技能 输入 做什么 产出 交互 触发条件
project-init 用户口述 引导式对话收集项目目标、交付物、成功标准、时间线、团队 project.yaml Project Owner 多轮对话 用户说"新建项目"
source-collection project.yaml 逐条阅读 2 周内邮件/Teams/会议 + 搜索 SharePoint & Web Sources/ + _index.md 无 project.yaml 创建后
todo-breakdown project.yaml + Sources/ meta-thinking 自审 → 拆解任务 → 分配 → 用户确认 todos.md Project Owner 确认方案 source-collection 完成后
project-heartbeat project.yaml + todos.md + 最近 2 条心跳 + 实时扫描（Teams、邮件、日历、会议） 扫描新内容 → 新增 Todo → 分配/启动/询问/验收 → 风险上报 更新 todos.md + Sources/ + heartbeats/*.md Member Agent 分配/追踪 · Project Owner 上报 每小时 工作日
project-dashboard project.yaml + todos.md + 最近 2 条心跳 生成项目状态全景报告 dashboard.md 无 心跳后 / 手动
todo-update project.yaml + todos.md + 心跳日志 + 广撒网扫描（Teams、邮件、日历、会议、Web、内部文档） 信息与目标碰撞 → 发现盲区和机会 → 更新 todos.md 更新 todos.md + heartbeats/*_todo-review.md Project Owner 确认废弃任务 每天
project-knowhow project.yaml + todos.md + 当天心跳 + knowhow.md 反思当天事件 → 提炼决策/教训/模式/人员洞察/工具 → ADD/REPLACE/EVICT knowhow.md（单文件迭代更新） 无 每天
member-heartbeat project.yaml + Sources/ + 项目 todos.md + 项目心跳 + member-todos.md + 成员心跳 + Owner 通信 + Project Agent 频道 深度思考 → 按状态分支：执行任务 / 根据反馈修改 / 交付已批准成果；向 Project Agent 提问，向 Owner 求助 更新 member-todos.md + deliverables/<task-id>/ + member-heartbeats/*.md Project Agent 提问/交付 · Human Owner Review/求助 每小时 工作日

04

API 与外部数据源

四个占位 API 处理 Agent 间及人机通信，八个 M365 和 Web 数据源提供项目情报。

通信 API

concordia_agent_send()

Project Agent → Member Agent

分配任务、询问进度、发送 Review 反馈、通知优先级变更、取消任务。

占位

escalate_to_owner()

Project Agent → Project Owner

风险上报、决策请求、废弃任务确认、关键阻塞。

占位

project_agent_comm()

Member Agent → Project Agent

提问澄清、报告进度、交付已批准成果（附完整文件路径）。

占位

owner_comm()

Member Agent → Human Owner

请求交付物 Review、遇到困难时求助、接收修改反馈或批准。
